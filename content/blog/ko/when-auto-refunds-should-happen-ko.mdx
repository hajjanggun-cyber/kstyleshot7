## 도입

환불 기준은 단순한 약관 문구가 아니라, 서비스 신뢰를 결정하는 핵심 장치다.

## 왜 중요한가

AI 생성 서비스는 결제, 큐 처리, 생성 실패가 얽혀 있기 때문에 사용자 잘못이 아닌 장애가 생길 수 있다.

## 자동 환불이 필요한 경우

### 1. 결제만 되고 작업이 시작되지 않은 경우

결제 성공 뒤 생성 파이프라인이 실제로 시작되지 않았다면 사용자가 비용을 떠안을 이유가 없다.

### 2. 시스템 오류가 복구되지 않는 경우

백엔드 실패로 결과물을 정상 전달할 수 없으면, 시간을 끌기보다 환불이 더 명확하다.

### 3. 서비스 측 원인으로 실패한 경우

결제 연동, 저장소, 생성 엔진 문제처럼 플랫폼 내부 원인이라면 자동 환불이 자연스럽다.

### 4. 공개한 정책에 이미 포함된 경우

한번 약속한 환불 조건은 예외 없이 일관되게 적용돼야 한다.

## 환불이 바로 되지 않는 경우

- 입력 셀카 자체가 부적절한 경우
- 결과가 취향과 다르지만 정상 생성된 경우
- 생성 완료 후 단순 변심인 경우

## 업로드 전 체크리스트

- 사용자 실수와 시스템 실수를 구분
- 환불 기준을 미리 공개
- 멱등하게 중복 환불 방지
- 실패 원인 기록 남기기

## 고지 문구

본 서비스는 특정 아티스트·기획사·브랜드와 제휴/공식 관계가 없습니다.

결과물은 AI로 생성된 스타일 이미지이며 동일 재현을 보장하지 않습니다.
