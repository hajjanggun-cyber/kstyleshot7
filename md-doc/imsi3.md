## 4:5 통일 메모

- 방향: 사용자 입력부터 헤어 합성, 헤어 결과, 최종 합성까지 전부 `4:5` 기준으로 통일하는 것이 맞음
- 이유:
  - 단계마다 비율이 다르면 크롭 기준이 계속 바뀜
  - 헤어 결과에서 괜찮던 구도가 최종 합성에서 다시 잘릴 수 있음
  - 사용자는 같은 이미지가 이어진다는 느낌보다, 단계마다 결과가 흔들린다고 느끼게 됨
  - 세로형 인물 중심 서비스 특성상 `4:5`가 가장 실용적임
- 원칙:
  - 업로드 직후부터 `4:5` 안전 프레임 사용
  - 헤어 모델 입력 이미지도 `4:5`
  - 헤어 결과 저장/미리보기도 `4:5`
  - 최종 합성도 `4:5`
  - 프론트 카드 비율도 `4:5`
- 현재 상태:
  - 최종 합성 `nano-banana-pro`는 `aspect_ratio: "4:5"` 명시됨
  - 헤어 합성 `change-haircut`은 현재 `aspect_ratio`를 보내지 않음
  - 즉, 현재는 모든 단계가 `4:5`로 통일된 상태는 아님

# Hair Flow UI Wireframe Proposal

Updated At: 2026-03-15 18:12 KST

## Goal

- 헤어 스타일 선택 UI를 옆으로 넘기는 탭형 구조에서, 한 화면에 전체 선택지를 볼 수 있는 섹션형 구조로 바꾼다.
- 사용자가 `내 사진 -> 생성 결과 비교 -> 스타일 선택 -> 컬러 선택 -> 다음 단계` 흐름을 바로 이해하게 만든다.
- 결과가 나온 뒤에는 샘플 탐색보다 `비교 후 선택`에 집중하도록 정보 우선순위를 재배치한다.

---

## Recommended Screen Order

1. 내 사진 스트립
2. 생성 결과 비교 영역
3. 헤어 스타일 선택 영역
4. 헤어 컬러 선택 영역
5. 하단 CTA 영역

## Core Design Direction

- 카테고리 전체를 한 화면에 모두 노출한다.
- `daily`, `performance`, `trendy`는 가로 탭이 아니라 독립된 섹션 카드로 보여준다.
- 각 카테고리 섹션에는 얇은 보더와 약한 배경색을 넣어 그룹 구분을 명확하게 만든다.
- 각 스타일 카드도 보더 기반 카드 UI로 정리한다.
- 선택된 카드는 두꺼운 강조 보더, 그림자, 선택 배지로 상태를 명확히 보여준다.
- 이미 생성한 스타일은 `생성됨` 또는 `Generated` 칩으로 상태를 보여준다.
- 생성 결과 카드 2개는 스타일 선택 영역보다 위에 두어 비교 행동을 우선시한다.

---

## Why This Structure Is Better

- 사용자가 현재 흐름을 즉시 이해할 수 있다.
- 옆으로 넘기지 않아도 전체 선택지를 한 번에 스캔할 수 있다.
- 결과 비교와 스타일 탐색이 섞이지 않아 선택 피로가 줄어든다.
- 결과가 나온 뒤에도 샘플 탐색 UI가 시선을 뺏지 않는다.
- 모바일에서도 섹션 단위로 읽기 쉬워서 구조 파악이 빠르다.

---

## Recommendation About Samples

- 결과가 아직 없을 때는 샘플 또는 스타일 선택 영역을 보여주는 것이 유효하다.
- 결과가 1개라도 생성된 이후에는 샘플의 우선순위를 낮추는 것이 좋다.
- 결과가 2개 준비된 시점에는 샘플을 계속 크게 보여주기보다, 결과 비교와 선택 행동에 집중시키는 편이 더 낫다.

권장 방향:

- 결과 없음: 샘플/스타일 탐색 중심
- 결과 1개 이상: 결과 비교 영역을 상단 우선 배치
- 결과 2개 완료: 샘플은 접거나 숨기고, 결과 선택이 최우선

---

## Suggested Visual Rules

- 바깥 섹션 카드: 연한 보더 + 약한 배경
- 스타일 카드 기본 상태: 얇은 보더 + 중립 배경
- 스타일 카드 선택 상태: 진한 보더 + 살짝 떠오르는 그림자
- 결과 카드: 스타일 카드보다 더 크게 배치
- 모바일: 2열 그리드
- 데스크톱: 3열 이상 확장

---

## Bottom Line

지금 구조보다 `전체 옵션을 한 화면에 노출하는 섹션형 카드 UI`가 더 낫다.

특히 헤어 단계는 샘플 탐색보다 `생성 결과 비교 -> 선택`이 더 중요한 단계이므로:

- 결과 영역을 `내 사진` 바로 아래에 배치하고
- 스타일 메뉴는 한 화면 전체 노출 구조로 바꾸고
- 카테고리와 카드에 보더를 넣어 정보 구획을 명확히 하는 방향이 적절하다.
