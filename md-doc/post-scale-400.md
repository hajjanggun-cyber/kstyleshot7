# kstyleshot Blog Scale Plan (420 Posts)

작성: Codex (GPT-5)
기준: EN 210개 + KO 210개
목표: 한글/영문 각각 200개 이상을 무리 없이 운영할 수 있는 7개 카테고리 체계 확정

## 운영 원칙

1. 카테고리는 7개로 고정하고, 세부 분류는 태그로 푼다.
2. 한글/영문은 번역 쌍이 아니라, 각 시장의 검색 의도와 전환 맥락에 맞게 독립 운영한다.
3. 카테고리 비중은 고정하되, 성과가 좋은 서브토픽만 2차 확장한다.
4. 우선순위가 높은 글에 더 긴 작성 시간을 쓰고, 낮은 우선순위 글은 템플릿 기반으로 빠르게 생산한다.
5. `post.md`의 120개 플랜은 초기 스타터 플랜으로 유지하고, 이 문서는 420개 규모 운영 기준으로 본다.
6. 실제 포스팅 추가는 `blog-seo-skeleton.md`의 고정 뼈대를 먼저 따르고 그 위에 누적한다.

## 1. 카테고리 확정 (7개)

고정 카테고리:

1. `Product / FAQ`
2. `Hair`
3. `Photo Technique`
4. `Beauty Prep`
5. `Outfit / Styling`
6. `Backdrop / Mood`
7. `Seasonal / Trend`

한글 운영명:

1. `제품 / FAQ`
2. `헤어`
3. `촬영 팁`
4. `K-뷰티 준비`
5. `코디`
6. `배경 / 무드`
7. `시즌 / 트렌드`

주의:

1. 카테고리는 7개를 넘기지 않는다.
2. `서울 배경`은 120개 플랜에서는 적절했지만, 420개 규모에서는 `배경 / 무드`로 넓혀야 확장이 쉽다.
3. 예: `홍대`, `성수`, `실내 스튜디오`, `야간 거리`, `궁궐 무드`, `강변 무드`는 전부 `배경 / 무드` 아래 서브토픽으로 넣는다.

1. `Hair`, `Photo Technique`, `Product / FAQ`가 실제 전환과 연결되는 핵심 축이다.

## 3. 각 카테고리별 서브토픽 구조

각 카테고리는 "클러스터" 단위로 운영한다. 각 클러스터에서 4~10개씩 파생하면 200개 이상까지 무리 없이 확장된다.

### 3.1 Product / FAQ

핵심 역할:

1. 가장 먼저 읽히는 글
2. 구매 전 불안 해소
3. 기능 기대치 조정

서브토픽 클러스터:

1. `셀카 업로드 기준`
   - 좋은 셀카 조건
   - 피해야 할 셀카 예시
   - 얼굴 각도/조명 최소 조건
   - 업로드 전 체크리스트
2. `서비스 사용 흐름`
   - 결제 -> 업로드 -> 선택 -> 다운로드
   - 왜 헤어가 먼저인지
   - 왜 단계형 UX인지
   - 단계별 실패를 줄이는 법
3. `기대치 조정`
   - 왜 원본 얼굴과 완전히 같지 않은지
   - 결과 편차가 생기는 이유
   - 배경 합성에서 기대할 것/말아야 할 것
   - 실제 생성형 도구의 한계
4. `정책 / 신뢰`
   - 환불 기준
   - 데이터 삭제
   - AI 처리 동의 관련 설명
   - 비제휴/비공식 안내
5. `문제 해결형 FAQ`
   - 결과가 어색할 때 가장 먼저 볼 것
   - 왜 업로드가 잘 안 먹히는지
   - 어떤 사진이 실패 확률이 높은지
   - 다시 시도할 때 바꿔야 할 한 가지

확장 방식:

1. 가장 성과 좋은 FAQ 글을 허브 글로 두고
2. 각 허브 글에서 촬영/헤어/코디 카테고리로 내부 링크를 뻗는다

### 3.2 Hair

핵심 역할:

1. 제품 핵심 기능과 가장 직접적으로 연결
2. 검색 의도와 클릭 유도가 강함

서브토픽 클러스터:

1. `앞머리`
   - 시스루뱅
   - 커튼뱅
   - 에어리 프린지
   - 무거운 앞머리 vs 가벼운 앞머리
2. `레이어 / 질감`
   - 허쉬컷
   - 소프트 레이어
   - 울프컷 계열
   - 부스스함 정리
3. `가르마 / 얼굴선`
   - 센터 파트
   - 사이드 파트
   - 옆머리 프레이밍
   - 턱선이 정리돼 보이는 헤어
4. `길이별 선택`
   - 숏
   - 미디
   - 롱
   - 중단발
5. `얼굴형별`
   - 둥근형
   - 긴형
   - 하관 강조형
   - 광대가 부각되는 경우
6. `초보자용 선택`
   - 실패 적은 헤어
   - 과한 스타일 피하는 법
   - 스타일 강도 조절
   - 셀카용으로 안전한 선택

확장 방식:

1. 기본형 글
2. 비교형 글 (`A vs B`)
3. 얼굴형 적용형 글
4. 업로드 전 체크리스트형 글

### 3.3 Photo Technique

핵심 역할:

1. 검색량이 넓고 롱테일 확보가 쉽다
2. 업로드 품질을 직접 개선한다

서브토픽 클러스터:

1. `카메라 각도`
   - 정면 셀카 각도
   - 눈높이
   - 턱 각도
   - 고개 각도
2. `거리 / 렌즈 왜곡`
   - 얼굴과 카메라 거리
   - 너무 가까운 셀카 문제
   - 광각 느낌 줄이기
   - 크롭 전제 촬영
3. `조명`
   - 창가 빛
   - 실내 조명
   - 역광 피하기
   - 노란 조명 보정
4. `구도 / 프레이밍`
   - 9:16 구도
   - 작은 방 셀카
   - 배경 정리
   - 상반신 프레이밍
5. `표정 / 자세`
   - 표정이 굳어 보일 때
   - 어색함 줄이는 법
   - 목/어깨 긴장 풀기
   - 눈 시선 처리
6. `업로드 전 체크`
   - 촬영 전 확인 리스트
   - 실패 확률 높은 실수
   - 셀카 재촬영 기준
   - 결과 비교법

확장 방식:

1. 아주 짧은 실전 팁 글을 많이 확보한다
2. Product / FAQ 허브로 계속 연결한다

### 3.4 Beauty Prep

핵심 역할:

1. "사진빨" 관련 의도를 끌어온다
2. 구매 전 설득형 콘텐츠로 적합하다

서브토픽 클러스터:

1. `피부결`
   - 셀카 전 피부 정리
   - 건조함 줄이기
   - 들뜸 줄이기
   - 과한 광 피하기
2. `베이스 메이크업`
   - 가벼운 베이스
   - 톤업 과다 방지
   - 광은 살리고 번들거림 줄이기
   - 저조도 메이크업 밸런스
3. `눈 / 눈썹`
   - 눈매 정리
   - 브로우 정돈
   - 음영 위치
   - 렌즈 컬러 선택
4. `립 / 블러시`
   - 립 컬러 선택
   - 과한 포인트 피하기
   - 자연스러운 혈색
   - 조명별 색감 차이
5. `문제 해결`
   - 다크서클
   - 붉은기
   - 번들거림
   - 피부가 뜨는 문제

확장 방식:

1. 과한 뷰티 리뷰형이 아니라 "사진 결과 개선형"으로 유지한다
2. 제품명 추천보다 원리 설명형이 안전하다

### 3.5 Outfit / Styling

핵심 역할:

1. outfit 기능이 붙었을 때 바로 자산이 된다
2. 스타일 탐색과 내부 링크 허브 역할을 한다

서브토픽 클러스터:

1. `실루엣`
   - 아우터 핏
   - 상의 핏
   - 구조감 있는 어깨
   - 상체 비율 정리
2. `넥라인 / 상반신`
   - 답답하지 않은 넥라인
   - 정면 셀카용 상의
   - 어깨선 정리
   - 얼굴과 옷의 밸런스
3. `컬러 조합`
   - 톤온톤
   - 무채색
   - 저채도 포인트
   - 배경과 색 매칭
4. `레이어드`
   - 니트
   - 재킷
   - 심플 레이어
   - 계절별 레이어드
5. `상황별 무드`
   - 도심 무드
   - 출근룩 계열
   - 에디토리얼 무드
   - 과한 코스튬 느낌 피하기

확장 방식:

1. outfit 기능 실연동 전에는 정보형 콘텐츠로 운영
2. 기능이 붙으면 선택 UX와 직접 링크 연결

### 3.6 Backdrop / Mood

핵심 역할:

1. kstyleshot만의 차별화 포인트
2. 헤어/코디와 결합해 조합형 콘텐츠가 가능함

서브토픽 클러스터:

1. `서울 무드`
   - 홍대
   - 성수
   - 한강
   - 궁궐 계열
2. `도심 / 거리`
   - 야간 거리
   - 번화가
   - 골목 무드
   - 미니멀 건축 배경
3. `실내 / 실외`
   - 카페 느낌
   - 갤러리 느낌
   - 스튜디오 느낌
   - 야외 개방감
4. `무드 조합`
   - 어떤 배경에 어떤 옷이 맞는지
   - 어떤 배경에 어떤 헤어가 맞는지
   - 배경 먼저 고를지 헤어 먼저 고를지
   - 얼굴이 묻히지 않는 배경 선택
5. `합성 기대치`
   - 복잡한 배경의 한계
   - 배경 합성에서 어색해지기 쉬운 경우
   - 결과를 더 자연스럽게 보이게 하는 선택

확장 방식:

1. 배경 단독 글
2. 배경+헤어 조합 글
3. 배경+코디 조합 글

### 3.7 Seasonal / Trend

핵심 역할:

1. 계절성 유입 확보
2. 캠페인성 발행 슬롯

서브토픽 클러스터:

1. `봄`
   - 화사한 톤
   - 가벼운 레이어
   - 밝은 배경
2. `여름`
   - 답답하지 않은 스타일
   - 밝은 조명 대응
   - 시원한 색감
3. `가을`
   - 레이어드
   - 톤다운 컬러
   - 분위기형 배경
4. `겨울`
   - 차가운 톤
   - 선명한 실루엣
   - 어두운 배경에서도 깔끔한 촬영
5. `트렌드성 변형`
   - 시즌 시작 체크리스트
   - 시즌별 실수
   - 시즌별 추천 조합

확장 방식:

1. 시즌 시작 전에 미리 발행
2. 해가 바뀌어도 재활용 가능한 evergreen 표현 위주로 작성

## 4. 발행 우선순위표 (중요도 + 시간 배분)

400개 이상 운영에서는 "글 수"와 "작성 시간"을 같은 비율로 쓰면 비효율적이다. 중요도에 따라 시간 배분을 따로 잡아야 한다.

### 4.1 티어 구조

| Tier | Priority | Post Share | Writing Time Share | Purpose |
|---|---|---:|---:|---|
| Tier 1 | 가장 먼저 | 20% | 40% | 전환 직결, 허브 글 |
| Tier 2 | 다음 | 35% | 35% | 유입 확장, 내부 링크 허브 보강 |
| Tier 3 | 후순위 | 45% | 25% | 롱테일, 변형, 보강 |

420개 기준 대략:

1. Tier 1: 약 84개
2. Tier 2: 약 147개
3. Tier 3: 약 189개

### 4.2 카테고리별 우선순위

#### Tier 1 (최우선)

1. `Product / FAQ`
2. `Hair`
3. `Photo Technique`

이유:

1. 실제 전환과 가장 가깝다
2. 검색 의도가 명확하다
3. 내부 링크 허브 역할이 강하다

권장 작성 밀도:

1. 글당 더 오래 쓴다
2. 구조/문장/CTA/내부 링크를 꼼꼼히 다듬는다
3. 카테고리 허브 글부터 먼저 만든다

#### Tier 2 (중요)

1. `Beauty Prep`
2. `Outfit / Styling`

이유:

1. 구매 직전 사용자의 망설임을 줄인다
2. 검색과 전환 사이를 연결해준다
3. 핵심 허브 글이 깔린 뒤 성과가 좋아진다

권장 작성 밀도:

1. 템플릿은 유지하되 사례와 연결 문장을 조금 더 넣는다
2. Tier 1 글로 내부 링크를 적극 연결한다

#### Tier 3 (보강)

1. `Backdrop / Mood`
2. `Seasonal / Trend`

이유:

1. 브랜드 차별화와 롱테일 확보에는 좋다
2. 하지만 초반 전환 효율은 핵심 카테고리보다 낮다

권장 작성 밀도:

1. 템플릿 기반으로 빠르게 생산한다
2. 조합형/파생형 글로 묶어서 배치 작성한다

### 4.3 추천 발행 순서

초기 운영 순서:

1. `Product / FAQ`
