## 문제 진단 요약

KO/EN 모든 글에서 공통으로 발생하는 3가지 패턴 문제.

### 문제 1 — 2문단 반복 패턴 (KO/EN 공통)

현황:
- `## 6-4 Pattern Diversification Rules`에 금지 규칙이 있음
- Codex가 H2마다 2문단을 반복하는 패턴을 계속 지킴
- 규칙이 "금지" 수준으로 적혀 있어도 실제 작성에서 어겨짐

원인:
- 규칙이 추상적 ("비슷한 길이의 문단 묶음 반복 금지")
- 구체적인 수치 기준이 없음

---

### 문제 2 — 실용 정보 누락 (KO/EN 공통)

현황:
- `## 6-5 Practical Info Blocks`가 "권장 규칙"으로 되어 있음
- Codex가 "선택"으로 해석하고 대부분 생략
- 장소형/방문형 글에서 입장료, 운영시간, 가는 법이 없음

원인:
- "권장"이라는 단어가 Codex에게 skip 신호로 작동

---

### 문제 3 — 중간 CTA 없음 (KO/EN 공통)

현황:
- `## 13 CTA 규칙`의 mid-CTA가 "0~1회 선택"으로 되어 있음
- Codex가 0으로 해석하고 항상 생략
- 하단 배너만 있고 본문 중간 링크 전혀 없음

원인:
- "optional"이 Codex에게 not required 신호로 작동

## post-codex-command.md 수정 내용

### 수정 1 — `## 6-4` 패턴 다양화 규칙 강화

금지 항목 아래에 아래 내용 추가:

```
[강제] H2별 문단 수 의무 다양화:
- 한 글 안에서 H2 아래 문단 수가 전부 같으면 반드시 깨야 한다
- 2문단 고정 패턴이 3개 이상 연속되면 그 다음 H2는 반드시 1문단 또는 3문단 이상으로 작성한다
- "이 H2는 왜 다른 길이인가"를 설명할 수 없으면 다양화가 안 된 것이다
```

---

### 수정 2 — `## 6-5` 실용 정보 블록 필수화

현재:
```
장소형 글, 방문형 글, 촬영형 글처럼 실용 검색 의도가 강한 주제는 본문 초반에 실용 정보 블록을 조건부로 넣을 수 있다.

권장 규칙:
```

변경:
```
장소형 글, 방문형 글, 촬영형 글은 본문 초반에 실용 정보 블록을 반드시 넣는다.

필수 규칙:
- 입장료/요금, 운영시간/휴무일, 가는 법 중 해당 주제에 맞는 항목을 1개 이상 반드시 포함한다
- bullet list 형식으로 본문 초반 H2 안에 배치한다
- KO와 EN 모두 동일하게 적용한다
```

---

### 수정 3 — `## 13` CTA 규칙 중간 링크 필수화

현재:
```
optional mid-CTA:
- mid-CTA 텍스트 링크는 0~1회만 선택적으로 사용할 수 있다
```

변경:
```
mid-CTA:
- 장소형/방문형/촬영형 글에서는 본문 중간에 텍스트 링크를 1회 반드시 넣는다
- 위치는 본문 전체 길이의 50~70% 지점 (마지막 H2 직전 또는 중간 H2 아래)
- KO와 EN 모두 동일하게 적용한다
- 문맥과 자연스럽게 이어지는 앵커 텍스트를 쓴다
- Quick Summary 직전 고정 삽입 금지 (위치 다양화)
- 공격적 판매 문구 금지
```
