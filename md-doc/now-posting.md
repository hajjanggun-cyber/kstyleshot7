## What To Change When Editing

### 1. Simple text-only edits

- Edit the target file in `content/hub/ko/[slug].mdx` or `content/hub/en/[slug].mdx`.
- Safe fields to update without changing URL:
  - `title`
  - `description`
  - `publishedAt`
  - `readTime`
  - `headerGradient`
  - `pullQuote`
  - article body text

### 2. If slug changes

- Change frontmatter `slug`
- Rename the file name to match the new slug
- Update all internal links pointing to the old slug
- Update `nextSlug` references in related articles
- Confirm the live URL changes to the new path

### 3. If hreflang pairing changes

- Keep `hreflangSlug` identical in both KO and EN files for the same topic
- If only one side changes, language toggle and alternate SEO links will break

### 4. If title or description changes

- These values are used for page metadata in `app/[lang]/hub/[slug]/page.tsx`
- Recount exact character length after editing
- Description must stay within `120-155` characters per project rule

### 5. If next article changes

- Update both `nextSlug` and `nextTitle`
- Confirm the target file actually exists

### 6. If image paths change

- Keep image files under `public/images/hub/[hub-slug]/`
- Use `.webp` paths in MDX
- Match locale suffix rules: KO `-kr.webp`, EN `-en.webp`

### 7. Structure rules that must remain valid

- H2 headings must start with `—`
- Insert `![]()` on the first line under each H2
- Keep Quick Summary box and related-links panel structure
- Keep locale-isolated internal links only
- Keep CTA banner at the bottom
  - KO: `/visuals/blog/blog-bottom-banner-kr.webp` -> `/ko`
  - EN: `/visuals/blog/blog-bottom-banner-en.webp` -> `/en`
