# image-webp-alt-prompt

이 파일은 앞으로 내가 먼저 읽어야 하는 고정 작업 규칙이다.

사용 방식:
- 너는 채팅에 이 파일만 먼저 던진다.
- 그다음 메시지에서 이미지 경로, ALT, 대상 URL, 삽입 위치만 준다.
- 나는 이 파일 규칙대로 바로 처리한다.

## 고정 규칙

- 사용자가 따로 말하지 않아도 KO 글과 EN 글을 둘 다 함께 처리한다.
- 이미지 파일명 끝이 `kr`이면 한국어 글에 넣는다.
- 이미지 파일명 끝이 `en`이면 영어 글에 넣는다.
- 한국어 글에는 한국어 ALT를 넣는다.
- 영어 글에는 영문 ALT를 넣는다.
- 이미지 경로는 본문에 `/image/...` 형식으로 넣는다.
- 실제 브라우저에서 보여야 하는 파일은 `public/image/` 기준으로 맞춘다.
- 원본 이미지가 너무 크면 용량을 줄이기 위해 WebP로 최적화한다.
- WebP 최적화 기준은 기본적으로 가로 960px 전후를 우선 사용한다.
- 본문 이미지가 모바일에서 과하게 크지 않게 유지한다.
- 본문 이미지는 가운데 정렬 상태를 유지한다.
- 사용자가 푸시를 따로 요청하지 않으면 깃푸시는 하지 않는다.
- 작업이 끝나면 어떤 파일에 어디에 넣었는지 짧게 보고한다.

## 네가 다음 메시지에서 주면 되는 정보

- 이미지 파일 경로
- 한국어 ALT
- 영문 ALT
- 대상 URL 또는 슬러그
- 어느 소제목 바로 위에 넣을지

## 내가 기본적으로 해석하는 방식

- 대상 URL이 `https://www.kstyleshot.com/ko/hub/...` 형식이면 KO 파일과 EN 파일을 같이 찾는다.
- 이미지 파일명이 `-kr`, `-en`으로 나뉘어 있으면 언어별 파일에 각각 맞게 연결한다.
- 사용자가 파일을 `public/image`에서 줬으면 그대로 쓰고, 본문에는 `/image/...`로 넣는다.
- 사용자가 WebP를 따로 말하지 않아도 파일이 크면 내가 최적화 여부를 판단해서 반영한다.

## 생략해도 되는 것

- `/image/...`로 넣으라는 말
- KO/EN 각각 넣으라는 말
- 모바일 크기 조정
- 가운데 정렬
- WebP 최적화 여부
- 푸시하지 말라는 말
